A language model only knows what it saw during training. It doesn't know your company's wiki, your latest emails, or anything that happened after its cutoff. **Retrieval-Augmented Generation (RAG)** bridges that gap.

## The pattern

1. **Store** your documents (split into chunks).
2. On a question, **retrieve** the most relevant chunks.
3. **Stuff** those chunks into the prompt as context.
4. The model answers using *that* context, not just its memory.

The win is twofold: the model can use fresh, private knowledge — and it's far less likely to **hallucinate**, because the answer is grounded in real retrieved text.

<Callout type="note">
Real systems retrieve by <em>meaning</em> using vector embeddings (remember vectors?). But the core loop — find relevant text, add it to the prompt — is exactly what you'll build below with a simple keyword match.
</Callout>

## Your turn — a tiny retriever

<CodeExercise
  id="rag-retrieve"
  prompt="Print the first document that contains the query word."
  starter={`docs = ["cats are mammals", "python is a language", "the sky is blue"]
query = "python"
# Print the first doc that contains the query word
`}
  expected="python is a language"
  solution={`docs = ["cats are mammals", "python is a language", "the sky is blue"]
query = "python"
print(next(d for d in docs if query in d))`}
/>

Swap keyword matching for embedding similarity and add the result to your prompt — that's a production RAG system in miniature.
