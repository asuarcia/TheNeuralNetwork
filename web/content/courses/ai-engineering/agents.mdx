A chatbot responds once. An **agent** pursues a goal over multiple steps: it decides what to do, uses a **tool** (search, code, an API), looks at the result, and decides again — until it's done.

## The agent loop

```
while not done:
    thought = model.plan(goal, history)   # decide next action
    result  = run_tool(thought.action)    # act in the world
    history.append(result)                # remember
    done = goal_met(history)
```

Three capabilities make it work:

- **Tools** — the agent can *do* things, not just talk (run code, query a DB, call an API).
- **Planning** — it breaks a goal into steps and picks the next action.
- **Memory** — it carries results forward so each step builds on the last.

<Callout type="warning">
Agents act in the real world, so guardrails matter: limit which tools they can call, cap the number of steps, and never give an agent more access than the task needs. (You'll see this again in the security course.)
</Callout>

## Your turn — a toy agent loop

<CodeExercise
  id="agent-loop"
  prompt="The agent doubles its value each step until it reaches the goal. Print how many steps it takes."
  starter={`value = 1
goal = 100
steps = 0
# Loop: while value < goal, double value and count the step. Print steps.
`}
  expected="7"
  solution={`value = 1
goal = 100
steps = 0
while value < goal:
    value *= 2
    steps += 1
print(steps)`}
/>

Real agents replace "double the value" with "call a tool the model chose," but the loop is identical. You've written the skeleton of an autonomous agent.
