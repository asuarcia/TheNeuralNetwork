Bag-of-words and one-hot vectors have a deep flaw: every word is equally far from every other. The model has no way to know "happy" and "joyful" are similar. **Word embeddings** solve this by giving each word a position in space where **closeness means similar meaning**.

## Meaning as coordinates

An embedding maps each word to a vector of numbers (often a few hundred), learned from huge amounts of text. The training trick (used by methods like **word2vec**) is simple: *words that appear in similar contexts get similar vectors.* "Tea" and "coffee" show up around the same words ("drink", "cup", "morning"), so they land near each other.

Now "similar meaning" becomes "nearby vectors," which we measure with the **cosine similarity** from the Math track. Identical direction → similarity 1.

<CodeExercise
  id="emb-cosine"
  prompt="Two word vectors point the same way: [1, 2] and [2, 4]. Compute their cosine similarity, rounded to 2 decimals (expect 1.0)."
  starter={`import math
a = [1, 2]
b = [2, 4]
# cosine = dot(a,b) / (length(a) * length(b)), rounded to 2 decimals
`}
  expected="1.0"
  solution={`import math
a = [1, 2]
b = [2, 4]
dot = sum(x * y for x, y in zip(a, b))
la = math.sqrt(sum(x * x for x in a))
lb = math.sqrt(sum(x * x for x in b))
print(round(dot / (la * lb), 2))`}
/>

## Meaning becomes arithmetic

The famous, almost magical property: directions in embedding space carry meaning, so you can do *analogies with arithmetic*. The classic example: **king − man + woman ≈ queen**. Subtracting "man" and adding "woman" moves along the "gender" direction.

<CodeExercise
  id="emb-analogy"
  prompt="Toy vectors: king=[5,5], man=[4,0], woman=[0,4]. Compute king - man + woman element-wise."
  starter={`king  = [5, 5]
man   = [4, 0]
woman = [0, 4]
# Print king - man + woman, element by element
`}
  expected="[1, 9]"
  solution={`king  = [5, 5]
man   = [4, 0]
woman = [0, 4]
print([k - m + w for k, m, w in zip(king, man, woman)])`}
/>

The result vector would land nearest to "queen" in a real embedding space. Relationships became geometry.

<Callout type="tip">
Embeddings aren't just for words. The same idea powers semantic search, recommendations, and the input layer of every transformer — text, images, and audio all get embedded into vectors where similar things sit close. This is one of the most important concepts in modern AI.
</Callout>
