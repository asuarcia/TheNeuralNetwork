Every impressive AI demo hides an unglamorous truth: most of the work is getting data into a usable shape. Garbage in, garbage out applies double to LLMs. A pipeline turns raw, messy sources into clean, structured text ready to embed or prompt.

## The stages of a pipeline

1. **Ingest** — pull raw bytes from somewhere: a file, a database, an API, a web page.
2. **Parse** — extract the actual text from its format (PDF, HTML, Markdown, JSON).
3. **Clean** — strip noise: whitespace, boilerplate, duplicates, empty records.
4. **Normalize** — standardize casing, dates, encodings so things compare equally.
5. **Chunk & embed** — split into pieces and vectorize (from the RAG lessons).

## Parsing structured formats

JSON is everywhere — API responses, config, exports. Python's `json` module turns a JSON string into a dictionary you can index.

<CodeExercise
  id="pipe-json"
  prompt="Parse the JSON string and print the value of the 'body' field."
  starter={`import json
raw = '{"title": "Welcome", "body": "hello world"}'
# Parse raw into a dict and print the body field
`}
  expected="hello world"
  solution={`import json
raw = '{"title": "Welcome", "body": "hello world"}'
doc = json.loads(raw)
print(doc["body"])`}
/>

<Callout type="note">
PDFs and HTML need dedicated parsers (like <code>pypdf</code> or <code>BeautifulSoup</code>) that aren't bundled here, but they all produce the same end product: a string of text you then clean and chunk exactly like this.
</Callout>

## Cleaning the mess

Raw text arrives with leading/trailing spaces, inconsistent casing, and empty entries. A cleaning pass fixes all of it in one comprehension — strip, lowercase, and drop the blanks.

<CodeExercise
  id="pipe-clean"
  prompt="Clean the lines: strip whitespace, lowercase, and remove any empty entries. Print the result."
  starter={`lines = ["  Hello ", "", "WORLD  ", "   "]
# Print a cleaned list: stripped, lowercased, no empty strings
`}
  expected="['hello', 'world']"
  solution={`lines = ["  Hello ", "", "WORLD  ", "   "]
print([l.strip().lower() for l in lines if l.strip()])`}
/>

## Normalizing and deduplicating

The same concept can appear as "AI", "ai", and "Ai". If you don't normalize, you'll store three copies and split your retrieval across them. Lowercasing then deduplicating collapses them into one.

<CodeExercise
  id="pipe-dedupe"
  prompt="Normalize tags to lowercase, then print the number of unique tags."
  starter={`tags = ["AI", "ai", "ML", "ml", "ai"]
# Print the count of unique, lowercased tags
`}
  expected="2"
  solution={`tags = ["AI", "ai", "ML", "ml", "ai"]
print(len(set(t.lower() for t in tags)))`}
/>

Five raw tags collapse to two real ones. That same move — normalize, then dedupe — keeps a knowledge base clean as it grows.

<Callout type="tip">
A good pipeline is <strong>idempotent</strong> and <strong>incremental</strong>: re-running it shouldn't create duplicates, and adding one new document shouldn't force you to re-process everything. Design for the second document, not just the first.
</Callout>
