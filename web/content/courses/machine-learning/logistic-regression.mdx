Linear regression predicts a number. But often you want a *category*: spam or not, fraud or legit, will-click or won't. **Logistic regression** adapts the exact same `w·x + b` machinery to predict a **probability** between 0 and 1 — and then a class.

## The problem with a raw line

A linear model can output anything: -40, 3.7, 9000. But a probability must live between 0 and 1. We need to squash that unbounded score into the `[0, 1]` range. Enter the **sigmoid** function:

```text
sigmoid(z) = 1 / (1 + e^(-z))
```

It takes any number and bends it into a smooth S-curve: very negative inputs → near 0, very positive → near 1, and `z = 0` → exactly 0.5.

## From probability to decision

Logistic regression computes `z = w·x + b` (the linear part), runs it through the sigmoid to get a probability, then applies a **threshold** — usually 0.5 — to decide the class. Above the threshold, predict "yes"; below it, "no". (That's the threshold idea from "Your First Model" in the Python for AI track, now with a principled probability behind it.)

A `z` of 2.0 gives a probability around 0.88 — comfortably a "yes."

## How it learns: log loss

Linear regression minimized squared error. Classification uses a different scorecard: **log loss** (cross-entropy). It punishes confident *wrong* answers brutally — predicting 0.99 for something that was actually "no" costs far more than predicting 0.6. That pressure pushes the model to be both correct *and* well-calibrated in its confidence.

The training mechanism is the same as everything else in ML: adjust `w` and `b` via gradient descent to drive the loss down.

<Callout type="tip">
Notice the pattern: <strong>linear core (w·x + b) → a function that shapes the output → a loss to minimize → gradient descent.</strong> Swap the shaping function and the loss, and you move from regression to classification — or, later, to a neuron in a neural network.
</Callout>
