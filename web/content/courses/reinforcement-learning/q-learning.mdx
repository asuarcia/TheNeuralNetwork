**Q-learning** is the classic RL algorithm, and it's beautifully simple: keep a table of how good each action is in each state, and nudge those numbers toward reality every time you act. It learned to play Atari, solve grid mazes, and balance poles.

## The Q-table

`Q(state, action)` estimates the **value of taking an action in a state** — the expected return that follows. Store these in a table: one number per (state, action) pair. At first they're all zero (the agent knows nothing). Through experience, they converge toward the truth.

To act, the agent looks up the current state's row and picks the action with the highest Q-value.

## The update rule

Here's the heart of it. After taking an action and seeing the reward and next state, nudge the old estimate toward a better one:

```text
Q(s,a) ← Q(s,a) + α · ( reward + γ · max Q(next state) − Q(s,a) )
```

- **α (alpha)** is the learning rate — how big a step to take.
- **reward + γ · max Q(next)** is the new, better estimate (immediate reward plus the discounted value of where you landed).
- The part in parentheses is the **error** — how wrong the old estimate was. (Sound familiar? It's gradient-descent thinking applied to values.)

<CodeExercise
  id="ql-update"
  prompt="Apply one Q-learning update. Q=0, alpha=0.5, reward=1, gamma=0.9, and the best next-state value is 0."
  starter={`Q = 0
alpha = 0.5
reward = 1
gamma = 0.9
max_next = 0
# Q_new = Q + alpha * (reward + gamma * max_next - Q). Print it.
`}
  expected="0.5"
  solution={`Q = 0
alpha = 0.5
reward = 1
gamma = 0.9
max_next = 0
print(Q + alpha * (reward + gamma * max_next - Q))`}
/>

Run this update over thousands of steps in a grid world, and the Q-table fills in until the agent reliably walks the best path to the goal. That's Q-learning.

<Callout type="tip">
Q-learning is <strong>model-free</strong>: the agent never needs to know the rules of the environment in advance. It learns purely from experienced (state, action, reward, next-state) transitions — which is why it's so widely applicable.
</Callout>

There's a catch: if the agent always takes the current-best action, it may never discover something better. Balancing trying-new-things against using-what-works is next.
