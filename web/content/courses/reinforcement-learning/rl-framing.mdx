The models so far learned from labeled examples. **Reinforcement learning (RL)** is different: there are no labels, just a goal and feedback. An **agent** tries things, gets **rewards**, and gradually learns which actions pay off. It's how AI mastered Go, Atari, robotics, and how chatbots get fine-tuned with human feedback.

## The core loop

RL is a conversation between an **agent** and an **environment**:

1. The environment shows the agent a **state** (the current situation).
2. The agent picks an **action**.
3. The environment returns a **reward** (good or bad) and the next state.
4. Repeat until the episode ends.

The agent's goal: choose actions that **maximize total reward** over time. Think of a game — the state is the screen, actions are button presses, reward is the score.

## Measuring success: total reward

An agent doesn't care about one step — it cares about the **total reward** over an episode (the *return*). Summing the rewards is how we score a run.

`}
  expected="2"

## Choosing actions

At each state the agent estimates how good each action is, and (usually) picks the best one. If it has value estimates for its options, choosing means taking the highest.

<CodeExercise
  id="rl-argmax"
  prompt="The agent estimates these action values. Print the action with the highest value."

# Print the action with the highest estimated value
`}
  expected="right"
  solution={`action_values = {"left": 1, "right": 3, "stay": 0}
print(max(action_values, key=action_values.get))`}
/>

<Callout type="tip">
The hard part of RL is that rewards can be <strong>delayed</strong>. A chess move's value isn't clear until the game ends. Figuring out which earlier actions deserve credit for a later reward — the "credit assignment problem" — is what the next lessons tackle.
</Callout>
