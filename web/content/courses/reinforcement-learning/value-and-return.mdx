An agent's whole job is choosing actions for **future** payoff. But a reward now isn't worth quite the same as a reward in ten steps. **Discounting** captures that, and leads to the single most important quantity in RL: **value**.

## The return, with discounting

The **return** is the total future reward from a point onward. But we shrink rewards that are further away using a **discount factor** called **gamma (γ)**, between 0 and 1. Each step into the future, the reward is multiplied by another factor of gamma:

```text
return = r₀ + γ·r₁ + γ²·r₂ + γ³·r₃ + ...
```

A gamma near 1 makes the agent far-sighted (cares about the long run); a gamma near 0 makes it greedy for immediate reward.

<CodeExercise
  id="rl-discounted"
  prompt="Compute the discounted return for rewards [1, 1, 1] with gamma = 0.5 (so 1 + 0.5 + 0.25)."
  starter={`rewards = [1, 1, 1]
gamma = 0.5
total = 0
# Add gamma**step * reward for each step. Print the discounted return.
`}
  expected="1.75"
  solution={`rewards = [1, 1, 1]
gamma = 0.5
total = 0
for step, r in enumerate(rewards):
    total = total + (gamma ** step) * r
print(total)`}
/>

The same three rewards are worth 3 undiscounted, but only 1.75 when later rewards count less.

## Value: expected return from a state

The **value** of a state is the *expected return* you'll get from there onward, if you play well. It answers: "how good is it to be *here*?" A state one move from winning has high value; a state about to lose has low value.

Value is the compass of RL. If you knew the true value of every state, you'd just always step toward the highest-value next state. The trouble is you *don't* know it up front — the agent has to **learn** it from experience. That learning algorithm is Q-learning, next.

<Callout type="tip">
"Value" connects straight back to the Math track's expected value: it's the average total (discounted) reward you expect from a state. RL is largely the art of estimating these values well from trial and error.
</Callout>
