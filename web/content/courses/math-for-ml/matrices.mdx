A **matrix** is a grid of numbers: rows and columns. In code, it's just a list of lists (each inner list is a row).

```python
M = [[1, 2],
     [3, 4]]
```

We describe a matrix by its **shape** — rows × columns. `M` above is 2×2. A dataset is a big matrix: one row per example, one column per feature.

## Transpose: flip rows and columns

The **transpose** turns rows into columns. It shows up whenever dimensions need to line up for a multiplication.

<CodeExercise
  id="mat-transpose"
  prompt="Transpose M (turn its rows into columns). Print the result."
  starter={`M = [[1, 2, 3],
     [4, 5, 6]]
# Print the transpose: [[1, 4], [2, 5], [3, 6]]
`}
  expected="[[1, 4], [2, 5], [3, 6]]"
  solution={`M = [[1, 2, 3],
     [4, 5, 6]]
print([list(col) for col in zip(*M)])`}
/>

## Matrix × vector: this *is* a layer

Here's the operation that makes deep learning tick. Multiplying a matrix by a vector means taking the **dot product of each row with the vector**. The result is a new vector — one number per row.

```text
[ 1  2 ]   [ 1 ]     [ 1·1 + 2·1 ]   [ 3 ]
[ 3  4 ] · [ 1 ]  =  [ 3·1 + 4·1 ] = [ 7 ]
```

Each row of the matrix is one neuron's weights; the output vector is the layer's activations before the bias and activation function. That's the whole reason matrices matter for AI.

<CodeExercise
  id="mat-vec"
  prompt="Multiply matrix M by vector v: take the dot product of each row with v. Print the result vector."
  starter={`M = [[1, 2],
     [3, 4]]
v = [1, 1]
# Print [dot(row0, v), dot(row1, v)]
`}
  expected="[3, 7]"
  solution={`M = [[1, 2],
     [3, 4]]
v = [1, 1]
print([sum(m * x for m, x in zip(row, v)) for row in M])`}
/>

<Callout type="tip">
A neural-network layer is one matrix multiply plus a bias and an activation. "10 neurons reading 4 inputs" is a 10×4 weight matrix. Training a network is just searching for the numbers inside these matrices.
</Callout>
