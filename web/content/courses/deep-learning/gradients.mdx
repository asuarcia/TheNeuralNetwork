We have a network that makes predictions and a loss that scores them. Learning means changing the weights to make the loss smaller. But there are millions of weights — how do we know which way to nudge each one?

## The gradient points uphill

For any weight, the **gradient** is the slope of the loss with respect to that weight: "if I increase this weight a little, does the loss go up or down, and how fast?"

To *reduce* loss, we step in the **opposite** direction of the gradient. That's **gradient descent**:

```text
w_new = w - lr * gradient
```

The **learning rate** (`lr`) controls step size — too big and you overshoot, too small and training crawls.

```python
w = 2.0
lr = 0.5
gradient = 1.0
w = w - lr * gradient
print(w)   # 1.5
```

Repeat this for every weight, over and over, and the network slides downhill toward weights that fit the data. **Backpropagation** is just the efficient bookkeeping that computes all those gradients at once.
