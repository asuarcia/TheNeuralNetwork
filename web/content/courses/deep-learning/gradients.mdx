We have a network that makes predictions and a loss that scores them. From the ML track, you know the update rule: step each weight opposite its gradient. But a deep network has *millions* of weights buried across many layers. How do we get the gradient for each one? The answer — **backpropagation** — is the algorithm that made deep learning practical.

## Recap: the update rule

For any weight, we step opposite its gradient (the slope of the loss with respect to that weight), scaled by the learning rate:

```text
w_new = w - learning_rate * gradient
```

print(w - lr * gradient)`}

Easy when someone *hands* you the gradient. The hard part is computing it for a weight that's buried five layers deep, feeding through everything after it. That's what backprop solves.

## The chain rule is the whole trick

A deep network is functions inside functions: layer 3 takes the output of layer 2, which took the output of layer 1. To find how an early weight affects the final loss, calculus gives us the **chain rule**: multiply the local slopes along the path.

If `y = (3x)²`, then `dy/dx = 2·(3x)·3` — the outer derivative times the inner derivative. Let's evaluate it.

<CodeExercise
  id="bp-chain"
  prompt="For y = (3x)², the chain rule gives dy/dx = 2*(3x)*3. Print dy/dx at x = 2."
  starter={`x = 2
# Chain rule: outer' (2*(3x)) times inner' (3). Print dy/dx at x=2.
`}
  expected="36"
  solution={`x = 2
print(2 * (3 * x) * 3)`}
/>

## Forward, then backward

Training each batch is two passes:

1. **Forward pass** — run inputs through the layers to get a prediction and the loss.
2. **Backward pass** — start from the loss and walk *backward*, using the chain rule to compute each layer's gradient from the one after it. The error signal flows from output toward input.

At each node, the gradient it sends back is its **upstream gradient × its local gradient**. That single multiply, applied node by node, propagates the error all the way to the first layer — hence "back-propagation."

<CodeExercise
  id="bp-node"
  prompt="A node receives an upstream gradient of 5.0 and its local derivative is 2.0. Print the gradient it passes to the previous layer."
  starter={`upstream = 5.0   # gradient arriving from the next layer
local = 2.0      # this node's local derivative
# Backprop multiplies them. Print the gradient passed backward.
`}
  expected="10.0"
  solution={`upstream = 5.0
local = 2.0
print(upstream * local)`}
/>

Chain those multiplications across every node and you've computed gradients for the *entire* network in one backward sweep — far cheaper than nudging each weight individually to see what happens.

<Callout type="note">
Real frameworks (PyTorch, TensorFlow) build a graph of these operations and run backprop automatically on GPUs — far too heavy for a browser tab. But "autograd" is exactly the chain-rule bookkeeping you just did by hand. The concept is the whole secret; the framework is convenience.
</Callout>

Forward to predict, backward to learn — that loop, run millions of times, is how every neural network on Earth is trained. Now let's assemble neurons, layers, and a forward pass into a complete network.
