A neuron's weighted sum `z` is linear. Stack a thousand linear steps and you still get... a line. To model curves, language, and images, networks need **non-linearity** — and that comes from the **activation function**.

## Why linearity alone fails

Composing linear functions gives you another linear function. `2(3x + 1) + 5` is still just a line. No matter how many linear layers you stack, the whole network collapses into a single line — useless for anything curved. The non-linear activation *between* layers is what breaks this collapse and unlocks real expressiveness.

## ReLU: the workhorse

The most common activation is almost insultingly simple — **ReLU** (Rectified Linear Unit): keep positives, zero out negatives.

```text
ReLU(z) = max(0, z)
```

```python
def relu(z):
    return max(0.0, z)

print(relu(3.0))    # 3.0
print(relu(-2.0))   # 0.0
```

That tiny "bend" at zero is enough: stack ReLUs across layers and the network can approximate almost any function.

## A zoo of activations

ReLU rules the hidden layers, but the **output** layer uses an activation matched to the task:

- **ReLU** — hidden layers; fast, simple, avoids some training problems.
- **Sigmoid** — squashes to (0, 1); used for binary probability (you met it in logistic regression).
- **Softmax** — turns a vector of scores into probabilities that sum to 1; used for multi-class output (which of 1,000 labels?).

A useful diagnostic: after ReLU, some neurons output exactly 0 — they're "inactive" for that input. Count them and you can see how sparse a layer's response is.

<CodeExercise

Only the two positive neurons fire; the negatives are silenced.

## Loss: the thing we minimize

To learn, the network needs a score for how wrong it is — the **loss**. You already met two: **MSE** for regression, and **cross-entropy** (log loss) for classification. Training is nothing more than nudging the weights to push the loss down, step by step.

<Callout type="tip">
Activation gives the network <strong>expressiveness</strong> (it can bend). Loss gives it a <strong>goal</strong> (get the number down). The next lesson connects them: how to compute which way to nudge every weight to reduce that loss — backpropagation.
</Callout>
