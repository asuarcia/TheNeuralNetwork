"Neural network" sounds intimidating until you see the unit it's built from. A single **neuron** does three small things: multiply, add, then squash.

## What a neuron computes

1. Multiply each input by a **weight**.
2. Add them all up, plus a **bias**.
3. Pass the result through an **activation function** (next lesson).

The first two steps are exactly the linear model you built last track — `w · x + b` — just with several inputs:

```text
z = (w1 * x1 + w2 * x2 + ...) + b
```

```python
inputs  = [1.0, 2.0]
weights = [0.5, -1.0]
bias    = 1.0

z = sum(i * w for i, w in zip(inputs, weights)) + bias
print(z)   # -0.5
```

<Callout type="tip">
A neuron is linear regression with multiple inputs, plus a squashing step. If you understood the last course, you already understand 80% of this one.
</Callout>

## Weights are what the network learns

The inputs are given. The **weights and bias are the knobs** — the parameters that training adjusts (via gradient descent, exactly like last track). A big positive weight says "this input matters a lot, in this direction." A near-zero weight says "ignore this input." Learning *is* finding good weights.

## Adding the activation

The raw `z` is linear. The neuron's final output applies an activation — here, ReLU (`max(0, z)`) — which is the "squash" step. That tiny non-linearity is what (next lesson) lets stacked neurons model curves.

Here `z = -0.5`, so ReLU clamps it to 0 — this neuron stays silent for this input.

## From a neuron to a layer

Put several neurons side by side, each with its **own** weights and bias, all reading the same inputs. That's a **layer** — it turns one input vector into a new vector (one number per neuron). Stack layers and you get a **deep** network. The math never gets harder than this; there's just a lot of it.
