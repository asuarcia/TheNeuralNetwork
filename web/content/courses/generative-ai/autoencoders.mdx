An **autoencoder** is a network that learns to copy its input to its output — which sounds pointless until you see the trick: it must pass everything through a tiny **bottleneck** in the middle. To rebuild the data from so few numbers, it's forced to learn the *essence* of it.

## Encoder, bottleneck, decoder

```text
input → [encoder] → latent code → [decoder] → reconstruction
        (compress)   (bottleneck)   (rebuild)
```

- The **encoder** compresses the input into a small **latent code** (a short vector).
- The **decoder** tries to reconstruct the original from that code.
- Training minimizes **reconstruction error** — how different the output is from the input.

Because the bottleneck is small, the model can't just memorize; it has to capture the meaningful structure. A 100-number image squeezed to 10 numbers means a 10× compression.

## Measuring reconstruction

How well did it rebuild the input? Same as always — squared error between input and output.

<CodeExercise
  id="ae-recon"
  prompt="Compute the reconstruction error (mean squared error) between the input and its reconstruction. Round to 2 decimals."
  starter={`inp   = [1, 2, 3]
recon = [1, 2, 2]
# Mean of (inp[i] - recon[i])**2. Print rounded to 2.
`}
  expected="0.33"
  solution={`inp   = [1, 2, 3]
recon = [1, 2, 2]
mse = sum((a - b) ** 2 for a, b in zip(inp, recon)) / len(inp)
print(round(mse, 2))`}
/>

## Latent space: where generation happens

That compressed code lives in a **latent space** — a compact map of the data where nearby points mean similar things. Here's the generative payoff: if you pick a *new* point in latent space and run it through the **decoder**, you get a *new* output that never existed. Move smoothly through latent space and the decoded images morph smoothly (a face slowly aging, a digit 3 becoming an 8).

A **Variational Autoencoder (VAE)** refines this so the latent space is smooth and well-organized, making it genuinely good for generating new samples.

<Callout type="tip">
"Latent space" is one of the most important ideas in generative AI. It's the same concept as embeddings: a compressed space where meaning is geometry. Modern image generators (latent diffusion) do their work in latent space precisely because it's smaller and more meaningful than raw pixels.
</Callout>
