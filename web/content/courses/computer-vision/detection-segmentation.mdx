So far our models answer "**what** is this image?" — a single label. Real scenes have many objects, and we often need to know **where** they are. That's the jump from classification to detection and segmentation.

## Three levels of vision

- **Classification** — one label for the whole image. *"This is a cat."*
- **Object detection** — find each object and draw a **bounding box** around it, with a label. *"Cat here, dog there."*
- **Segmentation** — label **every pixel**, tracing the exact outline of each object. *"These exact pixels are cat."*

Each level is more detailed — and more useful for things like self-driving cars (where is each pedestrian?) and medical imaging (which pixels are the tumor?).

## Bounding boxes are just coordinates

A detection box is four numbers — two corners — so its area is plain arithmetic.

## Measuring a detection: IoU

How do we know if a predicted box is *good*? We compare it to the true box with **Intersection over Union (IoU)** — the area where they overlap divided by the total area they cover together. IoU of 1.0 is a perfect match; near 0 means they barely overlap.

## How it's done (in brief)

Modern systems build on the CNN features you already understand:

- **Detection** — models like **YOLO** ("You Only Look Once") split the image into a grid and predict boxes + classes in a single fast pass, which is why they run in real time.
- **Segmentation** — models like **U-Net** (also used in diffusion!) shrink the image down to understand it, then expand it back up to label every pixel.

<Callout type="tip">
Notice the throughline: it's CNN features all the way down. Detection and segmentation add a "where" head on top of the same feature extractor you learned to build. Master convolution and the whole field opens up.
</Callout>
