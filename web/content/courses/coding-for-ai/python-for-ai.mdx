Every AI tool you'll build is, underneath, a Python program shuffling numbers around. You only need a handful of building blocks to follow along — and you'll write each one right here in the browser.

## Variables hold things

A **variable** is a labeled box. You put a value in, you take it out by name.

```python
learning_rate = 0.01
model_name = "tiny-net"
epochs = 10
```

## Lists hold many things, in order

A **list** is an ordered collection — exactly how we'll store datasets, predictions, and weights. Indexing starts at **zero**, and you can slice ranges.

```python
scores = [88, 92, 79, 95]
print(scores[0])    # 88   (first item)
print(scores[-1])   # 95   (last item)
print(scores[1:3])  # [92, 79]  (items 1 and 2)
```

<Callout type="tip">
Almost all data in machine learning is "a list of numbers" (or a list of lists). Get comfortable here and the rest feels familiar.
</Callout>

## Loops repeat work

A **for loop** walks through a collection one item at a time.

```python
for s in scores:
    print(s)
```

## Functions package up work

A **function** takes inputs, does something, and returns a result. A model is really just a (very fancy) function.

```python
def double(x):
    return x * 2

print(double(21))   # 42
```

## Dictionaries map keys to values

A **dictionary** stores labeled pairs — perfect for things like token→probability or word→count.

```python
prices = {"apple": 3, "banana": 1}
print(prices["apple"])   # 3
```

## Comprehensions transform whole lists

A **list comprehension** applies an expression to every item — you'll use these constantly to run a model over a batch of inputs.

`}
  expected="[2, 4, 6]"

print([x * 2 for x in nums])`}

## zip pairs two lists together

When you have predictions and truths — or inputs and weights — `zip` lets you walk both at once. This single pattern appears in nearly every lesson ahead.

<CodeExercise
  id="py-zip"
  prompt="Print a list of pairwise sums of a and b, e.g. [11, 22, 33]."
  starter={`a = [1, 2, 3]
b = [10, 20, 30]
# Print the element-wise sums of a and b
`}
  expected="[11, 22, 33]"
  solution={`a = [1, 2, 3]
b = [10, 20, 30]
print([x + y for x, y in zip(a, b)])`}
/>
