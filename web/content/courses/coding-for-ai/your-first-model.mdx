Forget neural networks for a moment. The simplest useful model is a **threshold rule**: if a number crosses a line, predict one thing; otherwise, predict the other. That's a real classifier, and spam filters started exactly here.

## The idea

Suppose something scores each email for "spamminess" from 0 to 1. We pick a **threshold** — say 0.5 — and decide:

```python
def predict(score, threshold=0.5):
    return "spam" if score > threshold else "not spam"

print(predict(0.9))   # spam
print(predict(0.1))   # not spam
```

That's it. The only "intelligence" is the threshold — and choosing a good threshold from data is what *learning* will do for us in later courses.

<Callout type="tip">
Every model, from this rule to GPT, is the same shape: <strong>input → some math → prediction</strong>. The models get fancier; the shape never changes.
</Callout>

## Run it on a batch

A model that handles one input is a demo. A model that handles a whole list is useful. Apply the rule across a batch and count the spam.

<CodeExercise
  id="model-threshold"
  prompt="Print how many scores are predicted spam (strictly greater than the threshold)."
  starter={`scores = [0.2, 0.9, 0.6, 0.1]
threshold = 0.5
# Print the count of scores predicted as spam
`}
  expected="2"
  solution={`scores = [0.2, 0.9, 0.6, 0.1]
threshold = 0.5
print(sum(1 for s in scores if s > threshold))`}
/>

## Now score it against the truth

Predictions are only meaningful next to reality. Each email here has a true label. Predict with the threshold, then count how many predictions were **correct** — that fraction is your model's *accuracy*.

<CodeExercise
  id="model-accuracy"
  prompt="Predict 'spam' if score > 0.5 else 'ham', then print how many predictions match the true labels."
  starter={`scores = [0.2, 0.9, 0.6, 0.1]
labels = ["ham", "spam", "spam", "ham"]
threshold = 0.5
# Print the number of correct predictions
`}
  expected="4"
  solution={`scores = [0.2, 0.9, 0.6, 0.1]
labels = ["ham", "spam", "spam", "ham"]
threshold = 0.5
correct = sum(
    1 for s, l in zip(scores, labels)
    if ("spam" if s > threshold else "ham") == l
)
print(correct)`}
/>

Four out of four — perfect on this tiny set. But notice: *we* picked 0.5. Slide the threshold and accuracy changes. A different threshold might catch more spam at the cost of flagging real mail.

<Callout type="note">
That tension — catch more bad stuff vs. annoy fewer real users — is the seed of precision and recall, which you'll measure properly in the Machine Learning track.
</Callout>
