In this project you'll make an AI hallucinate on purpose, then build a verification workflow that catches it. It's a hands-on lesson in why models invent facts — and how grounding defends against it. You'll use a real chat assistant for the generation and this notebook for the checking.

## 1. Generate a narrative with planted errors

<Callout type="tip">
**Do this now:** In your chat tool, ask for a confident, multi-paragraph summary of a historical event — and explicitly request that it include a couple of *subtle factual errors* (wrong dates, wrong names). Copy the result. You now have text that sounds authoritative but contains traps.
</Callout>

## 3. Check each claim against trusted sources

The core of the hunt: compare each claim to a set of trusted facts, and flag any that aren't supported. Count the unsupported ones — those are your hallucinations.

<CodeExercise
  id="hh-verify"
  prompt="Count how many claims are NOT supported by the trusted sources."
  starter={`trusted = {
    "The Eiffel Tower was completed in 1889.",
    "The Eiffel Tower is located in Paris.",
    "It was designed by Gustave Eiffel's company.",
}
claims = [
    "The Eiffel Tower was completed in 1889.",
    "The Eiffel Tower is located in Rome.",
    "It was designed by Gustave Eiffel's company.",
]
unsupported = sum(1 for c in claims if c not in trusted)
print(unsupported)`}
  expected="1"
  solution={`trusted = {
    "The Eiffel Tower was completed in 1889.",
    "The Eiffel Tower is located in Paris.",
    "It was designed by Gustave Eiffel's company.",
}
claims = [
    "The Eiffel Tower was completed in 1889.",
    "The Eiffel Tower is located in Rome.",
    "It was designed by Gustave Eiffel's company.",
]
unsupported = sum(1 for c in claims if c not in trusted)
print(unsupported)`}
/>

One claim ("located in Rome") isn't supported — caught. That exact-match check is crude, but it's the shape of every fact-checking system: extract claims, compare to a trusted source, flag the gaps.
