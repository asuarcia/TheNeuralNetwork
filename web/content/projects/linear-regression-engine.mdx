## 3. Measuring how wrong we are

To learn, the model needs a score. **Mean Squared Error** averages the squared gaps between predictions and truth. With the *right* weights (w=2, b=1) the error should be exactly 0.

<CodeExercise
  id="lr-cost"
  prompt="Write mse(w, b) over the data, then print the error for the perfect fit w=2, b=1."
  starter={`xs = [1, 2, 3, 4]
ys = [3, 5, 7, 9]

def mse(w, b):
    return sum((w * x + b - y) ** 2 for x, y in zip(xs, ys)) / len(xs)

# Print the error for the perfect fit
print(mse(2, 1))`}
  expected="0.0"
  solution={`xs = [1, 2, 3, 4]
ys = [3, 5, 7, 9]

def mse(w, b):
    return sum((w * x + b - y) ** 2 for x, y in zip(xs, ys)) / len(xs)

print(mse(2, 1))`}
/>

Try changing `2, 1` to `0, 0` and re-run — watch the error jump. That number is what training drives down.

## 4. The gradients

Gradient descent needs the slope of the error with respect to each weight. For MSE these are standard:

- `dL/dw = mean( 2 · (pred − y) · x )`
- `dL/db = mean( 2 · (pred − y) )`

<CodeExercise
  id="lr-grad"
  prompt="Compute the gradients at w=0, b=0 and print them rounded to 1 decimal."
  starter={`xs = [1, 2, 3, 4]
ys = [3, 5, 7, 9]
w, b = 0, 0

gw = sum(2 * (w * x + b - y) * x for x, y in zip(xs, ys)) / len(xs)
gb = sum(2 * (w * x + b - y) for x, y in zip(xs, ys)) / len(xs)
print(round(gw, 1), round(gb, 1))`}
  expected="-35.0 -12.0"
  solution={`xs = [1, 2, 3, 4]
ys = [3, 5, 7, 9]
w, b = 0, 0
gw = sum(2 * (w * x + b - y) * x for x, y in zip(xs, ys)) / len(xs)
gb = sum(2 * (w * x + b - y) for x, y in zip(xs, ys)) / len(xs)
print(round(gw, 1), round(gb, 1))`}
/>

Both gradients are negative — meaning we should *increase* w and b. That's exactly what the training loop does.

## 5. Train it

Now the payoff: loop the gradient-descent update many times and watch the model discover w≈2, b≈1 on its own.

<CodeExercise
  id="lr-train"
  prompt="Run gradient descent for 1000 steps and print the learned w, b (rounded to 1)."
  starter={`xs = [1, 2, 3, 4]
ys = [3, 5, 7, 9]
w, b = 0.0, 0.0
lr = 0.01

for _ in range(1000):
    gw = sum(2 * (w * x + b - y) * x for x, y in zip(xs, ys)) / len(xs)
    gb = sum(2 * (w * x + b - y) for x, y in zip(xs, ys)) / len(xs)
    w -= lr * gw
    b -= lr * gb

print(round(w, 1), round(b, 1))`}
  solution={`xs = [1, 2, 3, 4]
ys = [3, 5, 7, 9]
w, b = 0.0, 0.0
lr = 0.01
for _ in range(1000):
    gw = sum(2 * (w * x + b - y) * x for x, y in zip(xs, ys)) / len(xs)
    gb = sum(2 * (w * x + b - y) for x, y in zip(xs, ys)) / len(xs)
    w -= lr * gw
    b -= lr * gb
print(round(w, 1), round(b, 1))`}
/>

You built a model that learned from data — no libraries, just the math. 🎉
